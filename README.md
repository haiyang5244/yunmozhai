## 📖 项目简介

你是否遇到过这样的情况：一首诗读了很多遍，合上书却只记得开头几个字？或者某个字明明就在嘴边，却怎么也想不起来？

**拾珠落子·填字卡** 将背诵变成一场循序渐进的填字游戏。它随机抽取句子，挖去前后句的汉字，你需要从选字区中找出正确的字填入空白。每一次点击确认，系统会记录你的正确与错误，并智能调整后续出题难度——练得少、错得多的句子会优先出现，已记住的句子也会偶尔复习。从全模式到卡片模式，难度层层递进，陪你从生疏走向熟练。

## ✨ 主要功能

### 🎮 四种练习模式
| 模式 | 选字区内容 | 挖空范围 | 难度 |
|------|-----------|----------|------|
| **全模式** | 全部正确字 + 干扰字 | 中心句 + 随机一侧句子 | 辅助性强 |
| **半模式** | 一半正确字 + 干扰字 | 同上 | 需要手动补字 |
| **无模式** | 空（全部手动造字） | 同上 | 锻炼书写 |
| **卡片模式** | 空 | 整张卡片（只留一句提示） | 全文默写 |

### 🧠 智能出题
- 基于每个单句的**正确/错误次数**计算权重，优先抽取薄弱句子
- 已记住的句子权重降低但不会消失，防止遗忘
- 每句话独立记录，卡片集整体进度一目了然

### 📚 卡片集管理
- 主页最多同时展示 **3个** 卡片集（避免分心）
- 支持**归档**暂时不背的卡片集，随时移回
- 鼠标悬停查看卡片集的未练习/练习中/已记住数量
- 内置《唐诗二首》示例，开箱即用

### ✏️ 内容自定义
- 导入 `.txt` 文件，支持 `@name 书名` 格式定义卡片集名称
- 每张卡片以 `# 标题` 开头，正文按 `，。；？！` 自动分割单句
- 卡片详情页支持**编辑标题和正文**，编辑后会重置该卡片的练习数据
- 导出/导入全量数据备份（JSON格式）

### 🎨 交互细节
- **点击字块**自动填入第一个空缺格子，**点击已填字**即可移回选字区（无需拖拽）
- 造字输入框支持批量输入汉字，回车后自动生成字块
- 遮挡模式：在卡片详情页可以遮挡薄弱字，点击即可揭开
- 卡片列表支持按**顺序、遗忘程度、随机**三种排序
- 完成答题后会有颜文字反馈 `(๑•̀ㅂ•́)و✧` / `（´Д`）`

## 🚀 快速开始

### 在线使用
将项目部署在任何静态服务器上，或直接打开 `index.html` 文件即可运行（所有数据存储在浏览器本地）。

### 手动导入
1. 准备一个 `.txt` 文件，格式如下：
```txt
@name 诗经·国风

# 关雎
关关雎鸠，在河之洲。窈窕淑女，君子好逑。
---
# 蒹葭
蒹葭苍苍，白露为霜。所谓伊人，在水一方。
```
2. 在主页点击 `+` 号，选择该文件即可导入。

### 导出/导入备份
- 点击左侧悬浮球中的 **⚙️** 图标，选择“导出数据”即可下载当前所有卡片集及练习进度的JSON文件
- 选择“导入数据”可恢复备份

## 🛠️ 技术栈

- 纯原生 HTML5 / CSS3 / JavaScript (ES6)
- 无任何第三方依赖，轻量简洁
- 使用 localStorage 持久化数据
- 响应式设计，支持 PC 和移动端触摸操作

## 📁 数据格式说明

### 卡片集存储结构
```javascript
{
  "collections": [
    {
      "id": "唯一标识",
      "name": "卡片集名称",
      "archived": false,
      "cards": [ ... ],
      "summary": { "untried": 0, "practicing": 0, "mastered": 0 }
    }
  ],
  "currentMode": "full"   // full / half / none / card
}
```

### 单句练习数据
```javascript
{
  "correctCount": 0,
  "errorCount": 0
}
```
- 单句“已记住”条件：`correctCount >= 21 且 correctCount/3 > errorCount`（可调）
